# webdev_project
*Виконала Сачек Владислава КА-31*   
Мета веб-сайту: надати користувачу можливість переглядати опис книг, фільмів й серіалів, відкривати сторінки творів, виконувати базовий пошук та ознайомлюватися з контентом сервісу.   
   
Список створених сторінок:
1. Головна сторінка (Home)
2. Каталог (Catalog)
3. Сторінки окремих творів (Interstellar, Monster, Never Let Me Go)
4. Про сайт (About)
5. Контакти (Contacts)
6. Авторизація (Login)

На всіх сторінках використано стандартну структуру HTML5, семантичні теги, заголовки, абзаци, створене навігаційне меню. Також використані списки: неупорядковані ul для списків творів у каталозі, у Popular picks та для меню навігації; упорядковані ol у breadcrumbs на сторінках окремих творів, щоб показати послідовні переходи. Створені форми для пошуку, входу і реєстрації, зворотного зв'язку з використанням атрибутів валідації. Для фільтрів у каталозі були використані випадаючі списки <select>, для введення року та вибору жанрів – поле введення <input>, з типами number та checkbox відповідно.

CSS Flexbox використовувався для вирівнювання елементів у рядках і стовпцях, де важливою була адаптивність і динамічне вирівнювання. Зокрема, у шапці сайту для розміщення логотипу, навігації та поля пошуку, а також у картках контенту для вертикального вирівнювання зображення, назви та автора. Flexbox також використано для побудови бургер-меню на мобільних пристроях, де навігаційні елементи відображаються у вигляді вертикального списку.

CSS Grid був застосований для створення більш складних макетів сторінок. Наприклад, сітки карток на головній сторінці (у Popular picks) та у каталозі, що дозволяє автоматично адаптувати кількість елементів у рядку залежно від ширини екрану. Також Grid використано для сторінки каталогу з бічною панеллю фільтрів, де одна колонка відведена під фільтри, а друга — під основний контент.

Для забезпечення коректного відображення сайту на різних пристроях було використано media-запити. Вони дозволили змінювати структуру сторінок залежно від ширини екрана: на великих екранах навігація відображається у вигляді горизонтального меню, а на мобільних — перетворюється на бургер-меню, бічна панель фільтрів у каталозі на вузьких екранах переміщується зверху, зменшуються зовнішні відступи та розміри елементів для покращення читабельності на мобільних пристроях.
